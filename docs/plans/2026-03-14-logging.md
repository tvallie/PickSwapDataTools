# Logging Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add error-only rotating file logging to `fetch_nfl_players.py` so failures are captured persistently when the script runs unattended via launchd.

**Architecture:** Add a `RotatingFileHandler` logger at module level using Python stdlib `logging`. Patch the three existing error sites to call `logger.error()`. Wrap `main()` in a top-level try/except for uncaught exceptions. Terminal `print()` output is unchanged.

**Tech Stack:** Python 3 stdlib — `logging`, `logging.handlers.RotatingFileHandler`

### Task 1: Add logger setup at module level

**Files:**
- Modify: `fetch_nfl_players.py` (imports block + constants section)

**Step 1: Add imports**

In `fetch_nfl_players.py`, add to the existing imports block (after `import argparse`):

```python
import logging
from logging.handlers import RotatingFileHandler
```

**Step 2: Add logger setup below the constants block (after line 56, the `REMOTE_OUTPUT` line)**

```python
# ── Logging ──────────────────────────────────────────────────────────────────
_LOG_PATH = os.path.join(os.path.dirname(os.path.abspath(__file__)), "fetch_nfl_players.log")
_handler = RotatingFileHandler(_LOG_PATH, maxBytes=2 * 1024 * 1024, backupCount=5)
_handler.setFormatter(logging.Formatter("%(asctime)s  %(levelname)s  %(message)s"))
logger = logging.getLogger("fetch_nfl_players")
logger.setLevel(logging.ERROR)
logger.addHandler(_handler)
```

### Task 4: Patch error site 3 — top-level exception catch in `main()`

**Files:**
- Modify: `fetch_nfl_players.py` (the `main()` function and `if __name__ == "__main__":` block)

**Step 1: Wrap the `if __name__` block**

Find this code:
```python
if __name__ == "__main__":
    main()
```

Replace with:
```python
if __name__ == "__main__":
    try:
        main()
    except SystemExit:
        raise
    except Exception as e:
        logger.error("Unhandled exception: %s", e, exc_info=True)
        raise
```

> Note: `SystemExit` is re-raised without logging because it is intentional — it was already logged at its source in Task 2.
